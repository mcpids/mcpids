# Code of Conduct

MCPIDS is intended to be a serious, useful security project with a professional
and respectful contributor community.

## Expected Behavior

- Be direct, factual, and respectful in technical discussions
- Assume good intent, but back claims with reproducible details
- Welcome new contributors and help them find the right docs, tests, and code
- Discuss tradeoffs without personal attacks
- Handle security reports and sensitive user data carefully

## Unacceptable Behavior

- Harassment, discriminatory language, or personal insults
- Publishing private security details, credentials, or customer data
- Deliberate disruption of issues, PRs, or release workflows
- Spam, off-topic promotion, or impersonation
- Encouraging harmful deployment or misuse of security tooling

## Enforcement

Maintainers may warn, temporarily block, or permanently ban contributors who
violate this code of conduct. Maintainers may also lock issues or PRs if a
discussion becomes unproductive or unsafe.

To report a conduct issue, contact maintainers privately through the contact
information on the repository owner's GitHub profile. If that is not available,
open a low-detail issue requesting a private maintainer contact path.

## Scope

This code of conduct applies to repository issues, PRs, discussions, release
channels, and any project-managed community space associated with MCPIDS.
